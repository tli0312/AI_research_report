From Synchronous Chatting to Asynchronous Delegation

How Anthropic's Dispatch architecture, the "walkie-talkie" model, and a new generation of always-on agents are redefining human–AI interaction.

Section 01

The Paradigm Shift: Chat → Delegate

Between 2024 and 2026, the dominant mode of human–AI interaction underwent a fundamental transformation. What began as a turn-based, synchronous chat paradigm — the user types, the model responds, the user types again — has given way to an asynchronous delegation model where users assign tasks, walk away, and return to completed work.

Industry analysts identified three converging trends for 2026: the "Delegation Shift," the "Autonomous Workflow Loop," and the "Orchestration Awakening." The common thread is that humans stop being co-pilots and start being dispatchers.

Three Phases of Evolution

Phase 1 · 2023 – Mid 2024

Synchronous Chat

The ChatGPT-era baseline. The user sits at a keyboard, exchanges messages in real time, and the AI's usefulness is bounded by the session. Close the tab, lose the context. Each interaction is ephemeral and turn-based, requiring the human to remain actively engaged throughout.

Phase 2 · Mid 2024 – 2025

Agentic Augmentation

AI models gain tool use, code execution, and multi-step reasoning. Products like Claude Code, OpenAI's Operator (powered by CUA), and Google's Agent Development Kit let models browse the web, manipulate files, and chain actions. But the user still babysits: agents run inside a terminal or browser that must stay open. The paradigm is synchronous, with the human as active co-pilot.

Phase 3 · 2025 – 2026

Asynchronous Delegation

The leap. Users issue high-level goals ("refactor the auth module," "prepare the quarterly deck") and the AI agent works autonomously — across devices, across time, and across tools — notifying the human only at completion or critical decision points. The human's role shifts from operator to supervisor. If 2024 was the year of AI chatbots, 2026 belongs to AI agents.

Key Insight

The shift from "chat" to "delegate" is not merely UX polish — it represents a structural change in the compute topology. The user's attention is no longer a bottleneck in the execution loop. An agent that can accept work asynchronously and execute on a persistent local machine can accomplish tasks that are fundamentally impossible in a synchronous chat paradigm: overnight refactors, multi-hour data processing, and cross-application workflows that span hours.

Section 02

Case Study: Anthropic's Dispatch & the Remote Control Architecture

Anthropic has shipped a suite of complementary features that together form arguably the most complete implementation of asynchronous delegation currently available to consumers. Two are foundational: Claude Code Remote Control (February 24, 2026) and Claude Dispatch (March 17, 2026).

2.1   Claude Code Remote Control

Remote Control allows a developer to start a Claude Code session in a local terminal and then steer it from a phone, tablet, or any browser. The architecture is built on three principles:

Local execution, remote interface. The Claude Code process runs on the user's machine. It registers with the Anthropic API via outbound HTTPS, polls for work, and streams results back. No inbound ports are opened. All traffic travels through the Anthropic API over TLS — the same transport security as any Claude Code session — using multiple short-lived credentials, each scoped to a single purpose and expiring independently. The web and mobile clients are, in Anthropic's own words, "just a window into that local session."

Session persistence. If the laptop sleeps or the network drops, the session survives in the background and reconnects automatically when the host comes back online. An extended outage (~10 minutes without network) triggers a timeout. Conversation history stays synchronized across all connected devices — terminal, browser, and phone interchangeably.

Full environment access. Because execution is local, the agent retains access to the filesystem, MCP servers, project configuration, and installed tools — something cloud-hosted agents cannot replicate. Server mode (--spawn worktree) scales to 32 concurrent sessions, each in an isolated git worktree.

2.2   Claude Dispatch

Dispatch extends the Remote Control concept from developers to all Cowork (desktop) users. Where Remote Control requires a running terminal, Dispatch lets you message a task from the Claude mobile app and it spawns a Desktop session automatically to handle it. This creates what commentators have dubbed the "walkie-talkie" model: the phone is the command transmitter; the desktop is the execution receiver.

The key innovation is that Dispatch decouples intent from execution environment. The user formulates intent on whatever device is at hand (usually a phone), while compute-intensive, filesystem-dependent work happens on the more capable machine. A single, persistent conversation thread spans both surfaces.

Security by Design

Processing happens on the user's desktop. The phone sends only text instructions. Files stay local. No third-party data transfer. The sandbox uses default-deny networking with hard boundaries. The requirement that the Mac must be awake functions as an implicit security feature: no always-on cloud server means no always-on attack surface. Global Cowork instructions (e.g., "never delete files without confirmation") carry over to Dispatch sessions.

2.3   Solving the "Always-On Agent" Problem

The "always-on agent" problem has two dimensions:

Continuity: How does an agent maintain context and keep working when the user isn't looking? Locality: How does an agent access local files, tools, and credentials without shipping everything to the cloud?

Prior approaches forced a trade-off. Cloud-hosted agents solved continuity but sacrificed locality — they couldn't touch your local filesystem. Local agents preserved locality but required the user to sit at the machine. Dispatch resolves the tension by splitting the control plane from the execution plane: the agent runs locally (preserving file/tool access) but is controllable from anywhere via the API relay.

Section 03

Competitor Landscape: Who Else Has Built the Walkie-Talkie?

The local AI agent space exploded in early 2026. AI agent platform competition has crystallized into three distinct camps: proprietary integrated stacks (Anthropic, Meta/Manus), open-source local-first runtimes (OpenClaw), and enterprise orchestration platforms (NVIDIA, Google). Each embodies different trade-offs between accessibility, capability depth, and user control.

🦞
OpenClaw
Independent Foundation (ex-OpenAI backed) · MIT License

A self-hosted AI agent runtime that runs as a persistent background process on any device — Mac Mini, VPS, Raspberry Pi, or any Linux system. Launched November 2025 as "Clawdbot," it hit 9,000 GitHub stars in 24 hours and surpassed 214,000 by February 2026 — endorsed by Jensen Huang as "definitely the next ChatGPT."

Sessions are isolated per channel and per user, persisted to disk so the agent survives restarts without losing continuity. It answers on 20+ messaging channels (WhatsApp, Telegram, Slack, Discord, Signal, iMessage, Matrix, Teams, etc.) — a breadth Dispatch cannot match. Model-agnostic: can wire to Claude, GPT-4, local models, or any mix.

However, security researchers have flagged vulnerabilities including a zero-click exploit, and Cisco described the default configuration as a "security nightmare." The paradox: OpenClaw's low barrier to entry is exactly what prevents deep, secure use.

Open Source 20+ Channels Model-Agnostic Self-Hosted
🤖
Manus "My Computer"
Meta (acquired late 2025) · March 16, 2026

Launched one day before Dispatch. A desktop application for macOS and Windows that runs Manus's agent directly on user devices. Agents work with local files, tools, and applications through a CLI-first architecture — executing terminal commands directly on the machine.

The agent can utilize the computer's idle GPU to train ML models, turning an always-on desktop into a persistent AI assistant capable of accepting tasks remotely from a phone. Demos showed a working Swift app built in 20 minutes, no IDE opened. Runs on Meta's proprietary model stack for consistent quality, at the cost of a subscription.

Permission model: "Allow Once" for individual review, "Always Allow" for trusted recurring actions. The closest architectural parallel to Dispatch, though it routes through Meta's infrastructure.

Perplexity Personal Computer
Perplexity · March 11, 2026 · $200/mo Max tier

A "digital proxy" that works 24/7 on your behalf through a continuously running Mac mini. Unlike Dispatch or Manus, Perplexity's approach is hardware-coupled: the agent runs on dedicated Mac infrastructure, always on, always connected. Users orchestrate tools, tasks, and files from any device.

Instead of precise commands, users describe broad goals ("create an interactive educational guide") and the system determines which local apps to open, which files to use, and how to execute. Perplexity claims the enterprise version completed 3.25 years of work in four weeks.

All sensitive actions require user approval, every session generates a full audit trail, and there's a kill switch for emergencies. Mac-only at launch, with a waitlist for initial cohort.

Always-On Hardware Audit Trail Goal-Oriented Mac Mini
⚡
NVIDIA NemoClaw
NVIDIA · Unveiled GTC March 16, 2026 · Open Source

An open-source enterprise AI agent platform for autonomous task execution, built on NVIDIA's NeMo framework, Nemotron models, and NIM inference microservices. Positioned as the enterprise-grade complement to OpenClaw — "OpenClaw for the Fortune 500."

At its core is the NemoClaw runtime (also called OpenShell): a sandboxed execution environment with strict policy guardrails. Notably hardware-agnostic — runs on NVIDIA, Intel, AMD chips. Designed for companies to dispatch AI agents to perform tasks for their own employees.

Jensen Huang has reportedly pitched NemoClaw to Salesforce, Cisco, Google, Adobe, and CrowdStrike. The platform is focused on multi-step enterprise workflows rather than consumer phone-to-desktop control.

Enterprise Open Source Sandboxed Runtime Hardware-Agnostic

OpenAI CUA / ChatGPT Agent
OpenAI · Cloud-centric · Integrated into ChatGPT July 2025

OpenAI's Computer-Using Agent (CUA) powers Operator, which was fully integrated into ChatGPT as "agent mode" in July 2025. CUA combines GPT-4o's vision with reinforcement learning to interact with GUIs — clicking, scrolling, and typing like a human.

Performance: 38.1% on OSWorld (full computer use), 58.1% on WebArena, 87% on WebVoyager. The approach is cloud-centric — execution happens in OpenAI's infrastructure, not locally. No consumer-facing phone-to-desktop remote control analogous to Dispatch.

OpenAI's vision (per Sam Altman) is "proactive AI assistants constantly running in the background," but as of March 2026, this remains cloud-hosted without a local execution story.

Key capabilities: capability discovery through "Agent Cards" (JSON), task management with lifecycle states, agent-to-agent context sharing, and UI negotiation. Backed by 50+ partners including Salesforce, SAP, ServiceNow, and Workday. Now donated to the Linux Foundation.

Enterprise and multi-agent oriented rather than consumer-facing. No direct "phone controls your desktop" product, but provides the protocol plumbing others may build on.

Section 04

Technical Hurdles of Maintaining State Without Human Supervision

Maintaining state without constant human supervision is the central unsolved systems problem of the delegation era. The field of agent memory research is rapidly expanding but increasingly fragmented, with existing implementations differing substantially in their motivations, approaches, and evaluation protocols. The challenges cluster into five categories:

1

Context Persistence & Memory

AI agents are fundamentally stateless per invocation. Most agentic frameworks handle planning and orchestration but stop short of persistent memory. Without dedicated memory layers, agents restart from scratch each session, and developers compensate by feeding entire conversation histories into context windows — which scales poorly and wastes tokens on repeated context.

Storage-versus-inference trade-offs present a hard constraint: full history explodes costs, while aggressive summarization loses critical details. Emerging solutions include hierarchical memory with importance scoring and dynamic forgetting, external vector databases (Pinecone, Weaviate), Redis-backed state stores, and memory layer abstractions like Mem0. The 2025 "Agentic Context Engineering" (ACE) approach uses a three-agent loop — Generator, Reflector, Curator — to build and maintain a persistent context playbook. But none of these are standardized, and interoperability between frameworks remains poor.

Critical
2

Session Survival & Reconnection

Dispatch and Remote Control demonstrate one approach: keep the agent process alive locally and relay commands through a cloud API. But this creates fragility. If the host machine loses power, the session dies. Anthropic's current timeout is approximately 10 minutes of network loss before session termination. Perplexity's approach — dedicated, always-on Mac mini hardware — sidesteps this but introduces cost and hardware dependency.

Achieving true always-on behavior likely requires hybrid architectures where a cloud-side state snapshot can bootstrap a new local session after an outage — essentially a "warm standby" for agent state. OpenClaw's disk-persisted sessions survive restarts but still require the host process to be restarted manually.

High
3

Multi-Agent State Coordination

As agents increasingly collaborate (e.g., one gathers research while another writes a document), state persistence across agent boundaries becomes critical. Google's A2A protocol and Anthropic's MCP are early attempts at standardization, but production-grade solutions for shared mutable state across autonomous agents remain immature.

Inter-agent communication protocols, conflict resolution mechanisms, and orchestration logic become core challenges that did not exist in single-agent systems. The "drop-box" pattern (asynchronous knowledge sharing via written artifacts rather than real-time synchronization) shows promise — GitHub's Squad uses it for coordinated agents inside repositories — but scales poorly for tightly coupled workflows.

High
4

Security & Trust Without Supervision

An unsupervised agent with local filesystem access is a potent attack surface. The permission models are still rudimentary: Manus offers "Allow Once" / "Always Allow" toggles; Anthropic relies on workspace trust dialogs and sandboxing with default-deny networking; Perplexity provides audit trails and kill switches. NemoClaw introduces sandboxed execution with strict policy guardrails.

OpenClaw's zero-click exploit vulnerability illustrates the risk of open-source agents with weak default configurations. The industry has not yet developed robust capability-based security models — the equivalent of fine-grained Unix permissions for AI agent actions. The tension between usability ("Always Allow") and safety (per-action approval) remains unresolved for long-running autonomous tasks.

Critical
5

Error Cascades & Reliability Cliff

Without a human in the loop, agents can compound errors over long-running tasks. Models struggle with basic tool interactions, showing high failure rates due to formatting errors and parameter mismatches. The math is unforgiving: if each tool call has a 90% success rate, a 10-step workflow drops to 35% end-to-end reliability.

OpenAI's CUA benchmarks illustrate the gap: 38.1% success rate on OSWorld for full computer-use tasks — meaning the agent fails more often than it succeeds on complex workflows. The path to higher reliability involves better planning architectures, self-verification loops (the ACE "Reflector" pattern), and the ability to checkpoint and roll back to known-good states. Until then, "delegation" effectively means "supervised delegation."

Critical
The Reliability Threshold

For asynchronous delegation to become truly autonomous — where a user can fire-and-forget a complex task with confidence — agent reliability on multi-step workflows needs to climb from today's ~35-50% to above 90%. This likely requires a combination of better foundation models, specialized tool-calling fine-tuning, hierarchical planning with self-critique, and robust checkpoint/rollback infrastructure. The gap between "impressive demo" and "reliable production" remains the industry's defining challenge.

Section 05

Assessment & Outlook

The shift from synchronous chatting to asynchronous delegation is real and accelerating. The convergence of events in a single week of March 2026 — Perplexity's Personal Computer (March 11), NVIDIA's NemoClaw unveiling (March 16), Meta/Manus's My Computer (March 16), Anthropic's Dispatch (March 17), and Claude Code Channels (March 21) — represents an industry-wide phase transition, not a single company's product launch.

Architectural Verdict

Anthropic's Dispatch represents the most architecturally coherent implementation of the "phone as remote control, desktop as execution engine" model for several reasons: it keeps the relay minimal (no file storage in the cloud), the sandbox uses default-deny networking, and the existing MCP ecosystem provides a rich tool integration layer. Manus is the closest competitor in architectural terms, shipping near-identical capabilities within 24 hours, though it routes through Meta's broader infrastructure.

Perplexity takes a different bet — dedicated always-on hardware — which solves session survival at the cost of a $200/month price point and Mac lock-in. OpenClaw wins on breadth (20+ messaging channels, model-agnostic, free) but loses on security and polish. NVIDIA's NemoClaw is the enterprise play, and Google's A2A is the protocol play; neither is consumer-facing.

What Wins

The protocols that win this era will likely be MCP (for tool integration — already the de facto standard after broad industry adoption) and whatever emerges as the equivalent for inter-agent state sharing (A2A is the leading candidate). The interaction model that wins will be the one that most gracefully handles the transition between "I'm watching" and "I'll check back later" — and that is precisely the problem Dispatch is designed to solve.

The key technical bottleneck is not the command-and-control relay (which multiple vendors have now solved) but rather robust state persistence, error recovery, and security for long-running unsupervised tasks. Until agent reliability climbs well above 50% on complex workflows, the "delegation" model will remain "supervised delegation" — the human still needs to validate outputs before acting on them. But the direction is clear, and the infrastructure is being built at remarkable speed.
