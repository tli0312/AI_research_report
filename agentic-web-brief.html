Core Protocols

MCP + A2A: The Two-Protocol Foundation

MCP (Model Context Protocol) standardizes how agents invoke external tools via a JSON-RPC client-server model — the "USB-C of AI." Adopted by OpenAI, Google, and every major cloud provider. Donated to the Agentic AI Foundation (Linux Foundation) in December 2025. Limitations: no agent-to-agent communication, evolving security model, authentication delegated to transport layer.

A2A (Agent-to-Agent) solves the complementary problem: cross-framework agent collaboration. Agents publish JSON "Agent Cards" declaring capabilities; others interact via a standardized task lifecycle. v0.3 adds gRPC, signed Agent Cards, and Google Cloud Marketplace integration. The two protocols are explicitly complementary — MCP for tools, A2A for agents — and a well-architected system uses both.

Security & Governance

Critical Risks

Indirect prompt injection is the primary threat — malicious instructions in web content that agents ingest during normal operation. This is structural, not patchable: LLMs process instructions and data in the same context window. Agent impersonation exploits weak identity layers. Cascading multi-agent failures amplify single-point errors across delegation chains. Credential theft from agents holding OAuth tokens or wallet keys enables lateral movement across entire toolchains.

Governance-Containment Gap: 58% of enterprises monitor agent behavior, but only 37–40% have kill switches or purpose-binding controls. The industry can watch agents misbehave but often cannot stop them. NIST, the Agentic AI Foundation, and Cisco's 2026 security report are all working to close this gap — but production deployments are outpacing governance infrastructure.
Looking Ahead

Five Questions That Will Define the Next 12 Months

1. Will identity converge? Multiple incompatible standards risk replicating the federated-identity fragmentation of the human web. The IETF AIMS draft (composing existing standards) is the most promising convergence path.

2. Can prompt injection be structurally solved? If not, the agentic web may need a fundamentally new architecture separating reasoning context from data context. This is the single largest technical risk.

3. Who captures the economics? The value chain is shifting from human attention (ads) to agent capability (tool access, data quality). Websites optimizing for agent consumption may capture disproportionate value.

4. Open web or platform lock-in? Proprietary agent marketplaces and paywalled tool access could balkanize the agentic web. Protocol governance decisions being made now will determine the outcome.

5. How fast does regulation arrive? The EU AI Act will intersect with agentic infrastructure in ways not yet mapped. The gap between deployment speed and regulatory clarity creates both opportunity and systemic risk.
