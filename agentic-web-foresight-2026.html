Strategic Foresight Document — March 2026

The Agentic Web

A Predictive Analysis of How Autonomous AI Agents Will Reshape the Internet—2026 to 2036

Executive Summary

The internet is undergoing its fifth major platform shift. After the transitions from static pages (Web 1.0) to user-generated content (Web 2.0), from desktops to mobile, and from human-facing websites to the API economy, we are now entering the era of the agentic web—an internet increasingly navigated, transacted upon, and shaped by autonomous AI agents acting on behalf of humans and organizations.

This is not a speculative exercise. As of March 2026, the infrastructure is already materializing. Anthropic's Model Context Protocol (MCP) has become the de facto standard for connecting AI models to external tools, adopted by OpenAI, Google, and Microsoft. Google's Agent2Agent (A2A) protocol, now donated to the Linux Foundation, enables cross-vendor agent interoperability. NIST has launched its AI Agent Standards Initiative with active comment periods on agent identity and authorization. Gartner projects 40% of enterprise applications will embed AI agents by the end of 2026. And the first major agent security crisis has already arrived—the OpenClaw vulnerability affecting over 21,000 exposed instances demonstrated what cascading agent failures look like in practice.

Our core thesis: the agentic web will evolve through four distinct phases over the next decade, each defined not by a model capability breakthrough but by the maturity of surrounding infrastructure—identity, payments, liability frameworks, and trust protocols. The model intelligence is already sufficient. What constrains the agentic web today is the same thing that constrained e-commerce in 1996: the absence of trusted, standardized rails for authentication, transaction, and accountability. The analogy to SSL certificates and payment processors enabling the commercial web is direct and instructive.

The economic implications are profound. The advertising-supported, attention-economy model of the current web faces existential pressure when agents intermediate between humans and content. Value capture will shift from eyeballs to task completion, from clicks to outcomes. New business models—agent-native APIs priced on value delivered, capability marketplaces, reputation-as-a-service—will emerge. Some current internet giants are well-positioned (Google, Microsoft, Amazon); others face structural vulnerability (ad-dependent publishers, SEO-dependent content farms, traditional SaaS with human-only interfaces).

The transition will not be smooth. We identify seven critical inflection points, ranging from the adoption of universal agent identity standards (already in motion at NIST) to the first major agent-caused financial incident that triggers regulatory backlash. The EU AI Act's August 2026 enforcement deadline for high-risk systems will serve as the first significant regulatory test. The window between now and 2029 will determine whether the agentic web develops on open, interoperable standards or fragments into proprietary walled gardens—a question whose answer will shape the internet for decades.

1. Phased Evolution Model

The agentic web will not arrive as a single event but as a cascading series of capability expansions, each enabled by specific infrastructure and unlocked by a distinct killer application. We model this evolution in four phases, though the boundaries are gradational rather than sharp. Each phase persists alongside subsequent ones—tool-augmented copilots will not vanish when delegated task agents arrive, just as websites did not disappear when mobile apps emerged.

Phase 1

Tool-Augmented Copilots (Now–2027)

This is the current phase. AI agents can browse the web, call APIs, execute code, and interact with external tools, but they operate with explicit human approval for consequential actions. The user remains in the loop for every significant decision. OpenAI's ChatGPT with Operator merged in, Anthropic's Claude with MCP tool use, and Google's Gemini with agentic browsing in Chrome all exemplify this pattern. The agent is powerful but leashed.

Dominant Interaction Pattern
Human-directed with per-action approval. The human initiates, reviews, and confirms. The agent proposes and executes upon approval.
Technical Bottleneck
Reliability of multi-step execution. Current agents fail at roughly 15–30% of complex multi-step web tasks. Until reliability reaches 95%+ for routine workflows, humans cannot safely delegate.
Required Infrastructure
Standardized tool protocols (MCP—now achieved), browser-level agent integration (emerging in Chrome), basic OAuth-based agent authentication.
Killer App
Agentic coding assistants (Claude Code, Copilot, Cursor) that handle entire development workflows. This is where agents first prove they can reliably complete complex, multi-step tasks with measurable economic value.
Phase 2

Delegated Task Agents (2027–2029)

The critical transition: humans set goals and constraints, then walk away. The agent completes multi-step workflows autonomously within defined guardrails—booking travel itineraries, managing procurement workflows, conducting research and producing reports, handling routine customer service escalations. The human reviews outcomes rather than supervising each step.

Dominant Interaction Pattern
Goal-directed delegation with post-hoc review. Humans define objectives, budgets, and constraints. Agents execute autonomously and report results.
Technical Bottleneck
Agent identity, authorization, and liability frameworks. When an agent books a $3,000 flight on your behalf, who is legally liable if it books the wrong one? The identity and authorization infrastructure must be mature enough for services to trust agent-initiated transactions.
Required Infrastructure
Standardized agent identity credentials (NIST's initiative is the precursor), agent-native payment rails, robust audit logging, established legal frameworks for agent-initiated transactions.
Killer App
Autonomous personal procurement—an agent that manages your subscriptions, finds better deals, negotiates renewals, and handles routine purchasing within a budget you set. This is the "Mint.com moment" for the agentic web.
Phase 3

Agent-Native Services (2029–2032)

The web itself begins to restructure. When a significant percentage of "visitors" to a service are agents rather than humans, it becomes economically rational to design primarily for agent consumption. This is analogous to the mobile-first shift of 2012–2015, when websites were redesigned for phone screens because that's where the traffic was. Agent-native services will be API-first, semantically structured, machine-readable, and priced on value delivered rather than attention captured.

Dominant Interaction Pattern
Service-to-agent interfaces as the primary channel. Human UIs become secondary dashboards for oversight and exception handling, not the primary interaction surface.
Technical Bottleneck
Semantic interoperability at scale. Services must describe their capabilities in machine-readable formats that any agent can discover and consume without bespoke integration. The "agent app store" problem.
Required Infrastructure
Universal capability registries (like DNS for agent services), standardized semantic service descriptions, mature agent reputation systems, agent-native analytics and monitoring.
Killer App
Agent-mediated healthcare coordination—an agent that navigates insurance, schedules across multiple providers, manages prior authorizations, and tracks treatment plans. The complexity and friction of healthcare administration makes it the ideal domain for agent-native service redesign.
Phase 4

Agent Economies (2032+)

Agents transact with other agents at scale, with minimal human oversight for routine operations. Agent-to-agent marketplaces emerge for capabilities, data, and services. Specialized agents hire other specialized agents to complete subtasks, negotiate prices, and form ad hoc collaborations. Humans set high-level objectives and intervene only for novel situations or ethical edge cases. This is the fully realized agentic web.

Dominant Interaction Pattern
Agent swarms operating in multi-agent economies. Humans function as principals setting strategic objectives; agents function as autonomous economic actors within those mandates.
Technical Bottleneck
Trust and governance at machine speed. When agents negotiate and transact in milliseconds, human oversight becomes a throughput bottleneck. The system needs automated dispute resolution, machine-speed auditing, and robust principal-agent alignment guarantees.
Required Infrastructure
Agent-to-agent payment clearing, automated contract enforcement, real-time reputation scoring, machine-speed regulatory compliance checking, and mature principal-agent alignment mechanisms.
Killer App
Autonomous supply chain optimization—agent networks that manage global procurement, logistics, and inventory across thousands of suppliers, continuously renegotiating and rerouting in response to real-time conditions.

2. Critical Inflection Points

Seven inflection points will determine the trajectory and velocity of the agentic web's evolution. Each represents a discrete event or threshold that, once crossed, accelerates the transition to the next phase.

2.1 Universal Agent Identity Standard Adoption

NIST launched its AI Agent Standards Initiative in February 2026, and an IETF draft extending OAuth 2.0 for agent authentication is already in circulation. The preconditions are converging: MCP provides the tool-use layer, A2A provides the agent-to-agent communication layer, and now identity is the missing piece. We estimate a widely adopted standard by late 2027 to mid 2028, with medium-high confidence. If this arrives early, Phase 2 accelerates dramatically—agents can transact on behalf of users with legal backing. If delayed, the agentic web fragments into platform-specific identity silos, repeating the "Sign In with Google/Facebook/Apple" fragmentation of the 2010s but with higher stakes.

2.2 First Major Agent-Caused Financial Incident

The OpenClaw vulnerability of early 2026 was a security incident, but we have not yet seen an agent autonomously cause significant financial harm—a rogue trading agent, an agent that drains a payment account, or an agent procurement loop that generates six-figure charges. This is coming. We estimate a high-profile incident by mid 2027 with high confidence. The critical question is whether the regulatory response is proportionate (targeted safeguards, liability frameworks) or disproportionate (blanket restrictions on agent autonomy). The financial trading analogy is instructive: algorithmic trading caused the 2010 Flash Crash, leading to circuit breakers rather than a ban on algorithmic trading.

2.3 Browser-Level Agent Integration Becomes Standard

Google has already embedded agentic capabilities into Chrome with auto-browse. The precondition is that all major browsers follow suit, establishing a standardized browser agent API. We estimate this by late 2027 with medium confidence, as it depends on Apple's Safari strategy (Apple has historically been slow to adopt platform-disruptive web standards). If this arrives early, the "agentic browser" replaces the traditional browser as the default web access paradigm. If Safari remains a holdout, it creates a two-tier web experience reminiscent of the IE6 era.

2.4 EU AI Act Enforcement Creates Regulatory Precedent

August 2026 brings full application of the EU AI Act to high-risk AI systems, with penalties up to €35 million or 7% of global revenue. Many agentic applications—employment screening, credit scoring, healthcare triage—fall squarely into high-risk categories requiring human oversight under Article 14. The first enforcement actions will establish how strictly "human oversight" is interpreted for delegated task agents. We estimate the first significant enforcement action by Q1 2027 with high confidence. A strict interpretation that requires per-action human approval would effectively ban Phase 2 agents in the EU for regulated domains. A pragmatic interpretation that accepts post-hoc human review would accelerate adoption.

2.5 Agent-Native Payment Clearing Emerges

Current payment systems require human identity verification (KYC) that agents cannot satisfy. An agent-native payment layer—where agents hold delegated spending authority with cryptographic proof of authorization—is a prerequisite for Phase 3. We estimate initial deployments by 2028–2029 with medium confidence. Stripe, Square/Block (an AAIF co-founder), and potentially stablecoin-based systems are the likely providers. If this arrives via crypto rails, it could catalyze a revival of blockchain's utility narrative. If traditional fintech solves it first, it further marginalizes crypto's value proposition.

2.6 Agent App Store / Capability Marketplace Launch

A curated marketplace where agents can discover, evaluate, and purchase capabilities from other agents or services—the equivalent of Apple's App Store moment for the agentic web. This requires mature semantic service descriptions, reputation systems, and payment infrastructure. We estimate the first credible marketplace by 2028–2029 with medium confidence. The race is between platform incumbents (Apple, Google, Microsoft) and protocol-native upstarts building on MCP/A2A. Whoever captures this marketplace captures the most valuable chokepoint in the agentic economy.

2.7 Open-Source Agent Parity with Proprietary Models

If open-source agent frameworks (built on models like Llama, Mistral, or their successors) achieve parity with proprietary offerings from OpenAI, Anthropic, and Google, the economics of the agentic web shift fundamentally. Agents become commodity infrastructure rather than premium services. We estimate this inflection by 2028–2030 with medium-low confidence—the gap is currently widening for the most complex agent tasks even as it narrows for simpler ones. If open-source achieves parity, the value layer moves from model providers to orchestration and infrastructure. If it doesn't, the agentic web becomes an oligopoly controlled by 3–4 model providers.

3. The Agent-Native Web

3.1 How Websites and APIs Will Transform

When agents become a significant traffic source, the economics of web service design invert. Today's web is optimized for human perception: visual hierarchy, brand aesthetics, engagement loops. The agent-native web optimizes for machine comprehension: structured data, semantic markup, predictable API contracts, and deterministic behavior. This does not mean human interfaces disappear—they become a layer on top of agent-native infrastructure, just as mobile apps became a layer on top of REST APIs.

The transition will echo the mobile-responsive shift. Initially, services will add "agent endpoints" alongside human interfaces—a structured data feed that parallels the human-readable page. Over time, the agent endpoint becomes primary. Services that resist will face the same fate as businesses that refused to build mobile sites in 2013: invisible to a growing fraction of their potential market.

Rate limiting evolves from crude IP-based throttling to fine-grained, identity-aware policies. A verified agent with a high reputation score and an authorized payment method gets premium access; an anonymous scraper gets blocked. Pricing shifts from flat-rate subscriptions to value-based models: a real-estate agent-service might charge $0.50 per property listing retrieved by an agent, rather than $99/month for unlimited human browsing.

3.2 From SEO to AEO: Agent Engine Optimization

The transformation of search is already underway. Google's AI Mode now has over 75 million daily users, and agents are increasingly the intermediary between humans and information. Traditional SEO metrics—backlinks, domain authority, keyword density—matter less when an AI agent synthesizes information from multiple sources and presents a single answer. What matters instead is whether your content is structured, factually accurate, frequently cited by AI systems, and machine-parseable.

Agent Engine Optimization (AEO) will prioritize several factors: structured data markup (schema.org and beyond), content depth and factual density (articles over 2,900 words receive significantly more AI citations than shorter content), real-time data availability via API, and clear provenance and attribution chains. The SEO industry, currently a $100B+ global market, will undergo a painful transformation. Firms that adapt become AEO consultancies; firms that don't become irrelevant as agents bypass the traditional organic search funnel entirely.

For e-commerce, the implications are acute. Google has already launched agentic checkout with "Buy for me" functionality. When agents handle product discovery, comparison, and purchase, the merchant's brand relationship shifts from the consumer to the agent. If an agent cannot parse your inventory or price in real-time, you functionally do not exist in the agentic transaction layer.

3.3 New Agent-Native Service Categories

Several entirely new categories of service will emerge that have no human-facing equivalent. Agent capability registries will function as "DNS for skills"—directories where agents can discover what other agents or services can do, described in standardized semantic formats. Agent reputation services will track the reliability, accuracy, and trustworthiness of agents across interactions, analogous to credit scores for autonomous software. Orchestration-as-a-service providers will offer managed infrastructure for multi-agent workflows, handling the complexity of coordinating dozens of specialized agents toward a common goal. And context brokers will emerge as intermediaries that manage the rich personal context agents carry, providing privacy-preserving ways for agents to share relevant user preferences across services without exposing raw personal data.

4. Economic Model Shifts

4.1 The Attention Economy Under Siege

The current web's dominant business model—advertising funded by human attention—faces its most significant structural threat since the invention of the ad blocker. When an agent fetches information, compares products, or books services on a human's behalf, it does not see display ads. It does not have brand affinity. It does not impulse-buy. The entire edifice of the attention economy assumes a human eyeball at the other end of the HTTP request, and that assumption is eroding.

Google, to its credit, has already begun adapting. Sponsored placements now appear inside AI-generated responses in AI Mode, essentially paying for the agent to consider your product. This is the first iteration of "agent-mediated advertising"—paying not for human attention but for agent consideration. The CPM (cost per thousand impressions) model gives way to a CPA (cost per agent-considered action) model. But this transition compresses the value chain dramatically: instead of a funnel with awareness, consideration, and conversion stages (each monetizable), the agent collapses the funnel into a single decision point.

Publishers and content creators face an especially acute threat. If agents synthesize information without driving traffic to source websites, the ad-supported content model collapses. This is the "zero-click search" problem amplified by an order of magnitude. We expect significant legal battles over this issue by 2027–2028, echoing the news industry's conflicts with Google and Facebook over link taxation.

4.2 New Viable Business Models

The agentic web enables business models that are impractical or impossible today. Outcome-based pricing becomes viable when agents can measure and attribute results: a legal research service charges per actionable insight rather than per hour of access. Micro-negotiation services become possible when agents can negotiate on every transaction: a hotel that dynamically adjusts pricing in real-time based on agent-disclosed preferences and constraints. Agent-as-a-service platforms offer specialized, pre-trained agents that other agents can hire for subtasks—a tax calculation agent, a regulatory compliance agent, a translation agent—priced per invocation.

Perhaps most transformative: the agentic web dramatically lowers the barrier to launching a business. A single individual with a well-configured agent swarm can operate what previously required a team of 20—handling customer inquiries, managing procurement, coordinating logistics, maintaining the website, and optimizing pricing. The "solopreneur with agents" becomes a viable business structure at scales currently requiring formal organizations.

4.3 Winners and Losers

Well-Positioned: Microsoft (deep enterprise integration via Copilot, Azure infrastructure, LinkedIn data), Google (controls search, browser, Android, and cloud; already shipping agentic features), Amazon (AWS as agent hosting infrastructure, e-commerce as agent transaction target, Alexa as consumer agent interface), and Anthropic (MCP is becoming the TCP/IP of the agentic web; enterprise-first strategy positions them as the trusted platform). Apple is a wild card—its privacy-first approach and device control give it enormous leverage if it chooses to build agent infrastructure, but its historical reluctance to embrace open web standards could leave it behind.

Vulnerable: Ad-dependent publishers without differentiated data, traditional SaaS companies with human-only interfaces that resist API-first redesign, SEO agencies that fail to pivot to AEO, and consumer internet companies whose value comes from aggregating human attention (social media platforms face moderate risk, though their data moats provide some protection). Mid-tier cloud providers lacking AI capabilities face acquisition or irrelevance as the cloud layer increasingly requires native AI integration.

5. Second-Order Effects and Wildcards

5.1 Employment and Knowledge Work

The impact on knowledge work will be substantial but uneven. Customer service, data entry, basic research, routine legal work, and standard financial analysis will see 40–60% task automation by 2029. This does not mean 40–60% job loss—historical technology transitions show that task automation typically restructures roles rather than eliminating them wholesale. Customer service agents become "agent supervisors" managing AI agent fleets. Junior analysts become "agent trainers" who configure and validate autonomous research agents. The nature of the work changes more than the volume of employment, at least in the medium term.

Software development is the leading indicator. Agentic coding tools are already handling significant portions of routine development work, but the demand for developers who can architect agent systems, design agent-human interfaces, and manage agent reliability has surged. The net employment effect in software development has been expansionary so far, though the required skill profile has shifted dramatically.

5.2 Privacy in the Agent Era

Agents carrying rich personal context across services create a new category of privacy risk. Your agent knows your health conditions, financial situation, personal preferences, daily schedule, and social relationships—and it needs to share relevant slices of this information with services to act effectively on your behalf. The tension between utility (agents work better with more context) and privacy (more context means more exposure) becomes the defining challenge of the agentic web's trust architecture.

Emerging solutions include zero-knowledge proofs (proving you qualify for a discount without revealing your income), selective disclosure protocols (sharing only the minimum context needed for each transaction), and federated context management (your context stays on your device, with only derived decisions shared externally). The privacy implications also extend to the agent's own behavior being informative: observing which agents interact with which services reveals user behavior patterns even without accessing the underlying personal data.

5.3 Regulatory Landscape

Three regulatory philosophies are emerging. The EU's approach under the AI Act treats agents as high-risk systems requiring human oversight, transparency, and accountability—prescriptive and compliance-heavy. The US approach, currently led by NIST's standards initiative and existing executive orders, favors industry-led standards with voluntary compliance frameworks—permissive but potentially slow to address harms. China's approach emphasizes state control over agent capabilities and data access, with mandatory registration and content filtering—restrictive but enabling rapid deployment within controlled domains.

The regulatory divergence creates a fragmented global landscape. Multinational agent deployments will need to comply with multiple, potentially conflicting frameworks. This benefits large companies with compliance resources and disadvantages startups and open-source projects. We expect regulatory convergence to begin by 2029–2030, likely anchored by NIST standards and EU enforcement precedents, similar to how GDPR became the de facto global privacy baseline.

5.4 Wildcard Scenarios

Wildcard 1: Open-Source Agent Dominance. If open-source agent frameworks achieve and maintain superiority over proprietary offerings—plausible if the critical value layer shifts from model capability to orchestration and tooling—the economic structure of the agentic web looks radically different. Model providers become commodity infrastructure, value accrues to tooling and data, and the agentic web becomes as decentralized as the early web rather than as concentrated as the mobile app economy.
Wildcard 2: Catastrophic Agent Failure. A cascading multi-agent failure causes significant real-world harm—a supply chain agent network collapses, or autonomous financial agents trigger a market crash. Research already shows a single compromised agent can poison 87% of downstream decisions within four hours. A sufficiently dramatic incident could trigger a "Chernobyl moment" for agent autonomy, setting back Phase 2 adoption by years and empowering restrictive regulation.
Wildcard 3: Emergent Agent Coordination. Agents independently develop coordination behaviors their creators did not anticipate—implicit collusion in pricing, information asymmetry exploitation, or emergent "agent cartels" that prefer interacting with each other over alternatives. This is not science fiction: algorithmic pricing systems have already demonstrated implicit collusion in controlled studies. At agent scale, detecting and preventing such behavior becomes a novel regulatory and technical challenge.
Wildcard 4: The Agent Identity Crisis. If deepfake and impersonation technology outpaces agent identity verification, a crisis of trust could undermine the entire agentic web. If services cannot distinguish between a legitimate agent acting on a user's behalf and a malicious agent impersonating one, the system collapses to the lowest-trust equilibrium—every interaction requires burdensome human verification, negating the efficiency gains of agent delegation.

6. Historical Analogy Stress Test

6.1 Telephone Operators → Automated Switching

The replacement of human telephone operators with automated switching systems (1920s–1960s) is perhaps the closest historical analogy. Operators mediated every connection; automation made the network self-service. The transition took 40 years, was fiercely resisted by incumbent labor, and ultimately enabled a telecommunications revolution that operators could never have scaled to support. Where the analogy holds: the agentic web similarly replaces human intermediation with automated routing and execution. Where it breaks: telephone switching was a solved engineering problem with deterministic behavior; agent decision-making is probabilistic and context-dependent, introducing failure modes that telephone switching never had.

6.2 Algorithmic Trading

The evolution from human floor traders to algorithmic trading (1980s–2020s) offers the sharpest analogy for the economic dynamics of the agentic web. Algorithmic trading now accounts for 60–70% of equity market volume. The transition followed a familiar pattern: initial skepticism, a capability demonstration (algorithms consistently outperforming humans on speed and pattern recognition), rapid adoption, a major incident (the 2010 Flash Crash), proportionate regulation (circuit breakers, not bans), and eventual normalization. Where the analogy holds: the agentic web will likely follow the same pattern—incident, regulation, normalization. Where it breaks: algorithmic trading operates in a highly structured, rule-bound domain (financial markets); the agentic web operates across the entire unstructured internet, making failure modes more diverse and harder to anticipate.

6.3 Human Web Browsing → Programmatic API Access

The shift from humans manually visiting websites to programmatic API access (2005–2015) is the most direct precursor to the agentic web. APIs turned the web into a programmable platform, enabling mashups, integrations, and eventually the entire SaaS economy. Where the analogy holds: the agentic web extends this trajectory—agents are the next layer of abstraction above APIs, consuming APIs on behalf of humans. Where it breaks: APIs were designed for programmatic access from the start; much of the web that agents must navigate was designed for human browsers and has no API equivalent. This gap—the "unstructured web" problem—is why browser automation is a critical capability for current agents.

6.4 Human-Driven Cars → Autonomous Vehicles

The autonomous vehicle analogy is cautionary. Despite enormous investment and technical capability, full autonomy has been slower to deploy than predicted, largely due to edge cases, liability uncertainty, and public trust. Where the analogy holds: the agentic web faces similar challenges with edge cases and liability. Where it breaks critically: the cost of agent failure on the web is typically financial, not physical. An agent that books the wrong flight is annoying; an autonomous vehicle that hits a pedestrian is catastrophic. This difference in failure severity means the agentic web can tolerate a higher error rate, enabling faster adoption than autonomous vehicles. The regulatory bar is also lower for digital than physical autonomy.

Across all four analogies, the consistent pattern is: capability arrives before trust infrastructure, an incident forces the creation of governance frameworks, and adoption then accelerates under the new regime. We expect the agentic web to follow this pattern, with the incident phase occurring 2026–2028 and the post-incident acceleration occurring 2028–2032.

7. Convergence with Other Trends

7.1 Edge Computing and On-Device AI

On-device AI models (Apple's on-device intelligence, Qualcomm's NPUs, Google's Tensor chips) enable a class of agents that operate locally without cloud round-trips. This has profound implications for privacy (your agent processes sensitive context locally before selectively sharing with remote services) and latency (sub-second agent actions on mobile devices). The convergence of edge AI and the agentic web creates a "personal agent" paradigm where your phone runs a local agent that orchestrates cloud-based specialist agents, combining privacy with capability. By 2028, we expect on-device agent capabilities to handle 70–80% of routine agent tasks without cloud connectivity.

7.2 Blockchain and Decentralized Identity

Blockchain's strongest use case in the agentic web is not cryptocurrency but decentralized identity and verifiable credentials. Agent identity that doesn't depend on a single provider, verifiable capability credentials, and transparent reputation histories align naturally with decentralized architectures. The AAIF's inclusion of Block (formerly Square) as a co-founder and the exploration of stablecoin-based agent payment rails suggest this convergence is already being anticipated by major players. The timeline depends on whether decentralized identity solutions can achieve the usability and performance required for real-time agent interactions—a bar that blockchain systems have historically struggled to meet.

7.3 Spatial Computing and AR/VR

The intersection of agents and spatial computing creates persistent digital assistants that exist in your physical space. An AR agent that can see what you see, understand your physical context, and take action in both the digital and physical world represents the ultimate convergence of the agentic web with spatial computing. This is a 2030+ phenomenon, dependent on both agent maturity and AR hardware reaching mainstream adoption—both currently on uncertain timelines. The killer application here is agent-mediated physical world navigation: an agent that not only finds you the best restaurant but guides you there, checks you in, and handles payment as you arrive.

7.4 Robotics and Physical-World Agents

The agentic web's evolution will eventually merge with physical robotics, creating agents that operate across both digital and physical domains. A logistics agent that negotiates shipping rates (digital), dispatches a warehouse robot to retrieve goods (physical), and tracks delivery via IoT sensors (hybrid) represents the convergence endpoint. This is a Phase 4 phenomenon (2032+) and depends on advances in robotic dexterity and reliability that are currently on a slower trajectory than digital agent capabilities.

7.5 Quantum Computing

Within the 10-year horizon of this analysis, quantum computing's impact on the agentic web is likely limited to one critical domain: cryptography. If quantum computers break current encryption standards before quantum-resistant cryptography is widely deployed, the trust infrastructure of the agentic web (agent identity, secure communication, payment authorization) is at risk. Post-quantum cryptography migration is already underway, and we consider the risk of a quantum-induced agentic web disruption to be low within this timeframe, though it warrants monitoring.

What to Watch: Five Leading Indicators

The following five signals will indicate which scenario is unfolding and how fast the agentic web is maturing. Monitoring these provides the earliest possible signal of phase transitions.

1 Agent Traffic Share in Web Analytics. When major web analytics platforms (Google Analytics, Cloudflare Radar) begin reporting agent vs. human traffic as a standard metric, and when that share crosses 10%, 20%, then 30% thresholds for major services, each crossing marks a structural shift. The first services to report majority-agent traffic will be price comparison, travel booking, and financial data providers. Watch Cloudflare's annual internet reports for the first breakout of agent traffic data—this will be the clearest quantitative signal of Phase 2 arrival.

2 NIST Agent Identity Standard Progression. Track the progression from concept paper (current stage, comment period closing April 2026) to draft standard to final publication. The speed of this pipeline directly determines how fast the identity infrastructure for Phase 2 matures. If NIST moves at its typical pace, expect a draft standard by mid 2027. If industry pressure accelerates the process (as happened with post-quantum cryptography standards), it could arrive six months sooner—a significant acceleration for the entire agentic web timeline.

3 Agent-Related Litigation and Enforcement Actions. The first lawsuit over agent-caused harm, the first EU AI Act enforcement action targeting an agentic system, and the first insurance product specifically covering agent liability—each of these events signals that the legal system is beginning to grapple with agent autonomy. The nature of the regulatory response (proportionate vs. restrictive) will determine whether the agentic web accelerates or stalls in regulated industries.

4 Major Platform "Agent-First" Redesigns. Watch for the moment when a major web service (a top-50 website by traffic) publicly announces an "agent-first" redesign—restructuring their service primarily for agent consumption rather than human browsing. This is the "mobile-first" moment of the agentic web. We expect the first such announcement from a financial data provider, travel booking platform, or e-commerce marketplace by 2028. When this begins cascading across industries, Phase 3 has arrived.

5 Consumer Agent Adoption Curves. Track the adoption rates of consumer-facing agent products—ChatGPT's agentic features, Google's agent-enabled Search, Apple's eventual agent offerings. When a consumer agent product crosses the 100-million-active-user threshold with autonomous task delegation (not just chat), the mainstream inflection point for the agentic web has arrived. Google's AI Mode at 75 million daily users is close; the question is when these users begin delegating consequential actions (purchases, bookings, financial transactions) rather than just information retrieval.
