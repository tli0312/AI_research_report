The Agentic Web

Infrastructure, Protocols, and Competitive Dynamics for the Autonomous AI Internet — Q1 2026

Technology Infrastructure Research Report · March 2026

Executive Summary

A new infrastructure layer is forming beneath the surface of the internet. Just as HTTP, HTML, and DNS created the substrate for the human-readable web, a parallel stack of protocols, runtimes, identity systems, and payment rails is now being assembled to enable autonomous AI agents to navigate, transact, and collaborate across the open internet. We call this emergent layer the agentic web.

The pace of standardization in this domain has been remarkable. In the span of eighteen months—from late 2024 through early 2026—the industry moved from fragmented, proprietary tool-use APIs to two foundational open protocols: Anthropic's Model Context Protocol (MCP) for agent-tool interaction, and Google's Agent-to-Agent (A2A) protocol for inter-agent communication. Both have been donated to the Linux Foundation, and both enjoy adoption from every major cloud provider and AI lab. Meanwhile, NIST launched its AI Agent Standards Initiative in February 2026, the IETF published its first draft framework for agent identity (AIMS), and Stripe's Tempo blockchain went live with a native machine payments protocol. The agentic web is no longer speculative; it is being built in production.

Yet critical gaps remain. Agent identity and authorization lack a unified standard. Payment infrastructure is bifurcated between crypto-native and traditional rails. Observability, auditing, and reversibility of agent actions are largely unsolved at scale. And the security implications of prompt injection through web content represent a structural vulnerability that no framework has fully addressed. The winners of the next phase will be those who close these gaps while the protocols are still consolidating.

Key Findings
  • MCP has reached over 10,000 active public servers and 97 million monthly SDK downloads, establishing it as the de facto standard for agent-tool interaction.
  • The A2A protocol (v0.3) now has support from 150+ organizations and is being integrated into Google Cloud's Agent Marketplace.
  • Agent payment infrastructure is bifurcating: Stripe's Machine Payments Protocol (MPP) handles enterprise fiat flows; the x402 protocol enables permissionless crypto micropayments. Over 35 million transactions have flowed through x402 as of March 2026.
  • Agentic AI companies have raised $20.8 billion cumulatively, with $5.99 billion in 2025 alone—a 30% year-over-year increase. Seed rounds for "agentic" startups command a 40% valuation premium over generic generative AI tools.
  • NIST, the IETF, and the Linux Foundation are all actively developing governance and interoperability standards, signaling that the agentic web is entering its institutionalization phase.
Section 1

Defining the Agentic Web

The concept of the agentic web is best understood through contrast with its predecessors. The traditional web (Web 1.0/2.0) was designed for human consumption: browsers render pages, users click links, and interaction is mediated by visual interfaces. The semantic web (Tim Berners-Lee's vision, partially realized through RDF, OWL, and linked data) attempted to make web content machine-interpretable by adding structured metadata—but it assumed deterministic parsers, not probabilistic reasoning engines. The agentic web departs from both by assuming that the primary consumer of web-exposed services will increasingly be an autonomous AI agent—an entity capable of reasoning about goals, selecting tools, managing context across sessions, and transacting on behalf of a human principal.

Core Primitives

The agentic web requires a set of primitives that the traditional web never needed. Identity: not just "who is this user?" but "who is this agent, who authorized it, and what are its permission boundaries?" Authentication and delegation: mechanisms for a human to grant an agent scoped authority to act, with clear audit trails. Tool use: a standardized interface for agents to discover and invoke external capabilities—APIs, databases, code execution environments—without per-service custom integration. Inter-agent communication: protocols for agents built on different frameworks, by different vendors, to discover each other's capabilities and collaborate on tasks. Payment: infrastructure for agents to autonomously pay for services, with spending limits, authorization chains, and settlement. Memory and state: persistent context that survives individual sessions, enabling agents to build up knowledge and track multi-step workflows over time. Observability: the ability to monitor, audit, and replay agent actions for debugging, compliance, and trust.

Relationship to Adjacent Concepts

The agentic web intersects meaningfully with several adjacent movements. Web3 and decentralized identity provide some of the cryptographic primitives (DIDs, verifiable credentials, on-chain attestations) that agent identity systems are now borrowing—though the agentic web is emphatically not limited to blockchain-based approaches. API-first architectures have laid essential groundwork by making functionality available programmatically, but the agentic web goes further by requiring semantic discovery (agents need to understand what an API does, not just its endpoint schema) and dynamic composition (agents assemble tool chains at runtime based on goals). The distinction matters: the API economy assumed human developers as intermediaries; the agentic web assumes AI agents operating with minimal human supervision.

Agent Runtimes and Frameworks

The framework layer is the most mature segment of the agentic stack, though it remains highly fragmented. LangChain/LangGraph dominates by ecosystem size, with 47 million+ PyPI downloads and over 600 integrations spanning every major LLM, database, and tool provider. LangGraph's approach—representing workflows as directed graphs with nodes and edges—provides the structural precision needed for complex multi-step agent workflows. CrewAI has emerged as the fastest-growing framework for multi-agent use cases, with an intuitive role-based mental model: define agents with roles, assign goals, organize them into crews, and let them collaborate. Microsoft AutoGen (AG2) pioneered the multi-agent conversation paradigm but has seen slowing development momentum. OpenAI's Agents SDK offers the lowest barrier to entry and tight integration with GPT-4/GPT-5 models, making it the default for teams already embedded in the OpenAI ecosystem. Google's Agent Development Kit (ADK), released alongside Vertex AI Agent Builder, emphasizes open interoperability and native support for both MCP and A2A protocols.

Tool-Use Protocols and Web Discovery

The tool-use layer has consolidated rapidly around MCP (discussed in depth in Section 3). Alongside MCP, a parallel evolution is occurring at the web surface layer—new file standards that make websites agent-readable. llms.txt, proposed by Jeremy Howard in September 2024, is a Markdown file served at a site's root that provides LLM-optimized summaries of a site's content and capabilities. Adoption has been growing steadily, with reports of 30–70% reductions in hallucination rates when models can reference an authoritative llms.txt. agenticweb.md goes further, providing a semantically rich, machine-readable interface for autonomous agents. The Agent Web Protocol takes a more structured approach, defining an agent.json file that declares available actions, authentication methods, and rate limits—essentially a robots.txt for the agentic era. These are complementary rather than competing: llms.txt optimizes content retrieval, while agenticweb.md and agent.json enable action and transactional capabilities.

Critical Gaps

Two layers stand out as dangerously under-built relative to their importance. Agent identity and authorization remains the most critical gap. No single standard has emerged for representing an agent's identity, delegation chain, or permission boundaries in a way that is portable across frameworks and platforms. Microsoft Entra Agent ID, the OpenAgents AgentID system, and the IETF's AIMS draft all propose different approaches, and the industry has yet to converge. Observability and governance is similarly immature: while 58–59% of enterprises report some form of agent monitoring, only 37–40% have implemented actual containment controls like purpose-binding or kill-switch capabilities. This governance-containment gap represents a systemic risk as agents are deployed in higher-stakes environments.

Section 3

Protocol Deep Dives

Model Context Protocol (MCP)

MCP, announced by Anthropic in November 2024, has become the defining protocol of the agentic web's first phase. Its architecture follows a client-server model: an MCP client (embedded in an agent or LLM application) connects to MCP servers that expose tools, resources, and prompts through a standardized JSON-RPC interface. The protocol's power lies in its simplicity and universality—just as USB provided a single connector for peripherals, MCP provides a single protocol for AI tool integration.

The adoption trajectory has been extraordinary by any standard. OpenAI adopted MCP across its products in March 2025. Google DeepMind followed. By mid-2025, MCP's dominance was clear: over 10,000 active public servers, 97 million monthly SDK downloads, and deployment across Fortune 500 enterprises. In December 2025, Anthropic donated MCP to the newly formed Agentic AI Foundation under the Linux Foundation, with founding members including Block, OpenAI, AWS, Google, and Microsoft. This move transformed MCP from a single company's project into community-governed infrastructure.

MCP's strengths are clear: massive ecosystem, vendor neutrality (post-donation), simple mental model, and real-world production validation. Its limitations are equally important. MCP is fundamentally a tool-use protocol—it standardizes how an agent invokes an external capability but does not address how agents discover each other, negotiate tasks, or manage multi-party workflows. Additionally, MCP's security model is still evolving; the protocol relies on the transport layer for authentication, which can be insufficient for high-trust enterprise environments. The MCP roadmap for 2026 includes enhanced authorization flows, streaming capabilities, and richer resource typing—all needed for the protocol to mature from a developer tool into enterprise infrastructure.

Agent-to-Agent Protocol (A2A)

Google introduced A2A in April 2025 to address the complementary problem that MCP does not solve: how do agents, potentially built on different frameworks by different organizations, communicate with each other? A2A's design philosophy centers on treating agents as opaque services. An agent need not expose its internal architecture or model; it publishes an Agent Card (a JSON descriptor of its capabilities, input/output schemas, and authentication requirements), and other agents interact with it through a standardized task lifecycle: create task → send messages → receive results.

Version 0.3, released in early 2026, brought significant upgrades: gRPC support for high-throughput use cases, cryptographic signing of Agent Cards for integrity verification, and extended Python SDK capabilities. The protocol now has support from over 150 organizations and has been integrated into Google Cloud's AI Agent Marketplace, where ISVs and system integrators can publish A2A-compatible agents for direct purchase. The Linux Foundation has also taken governance of A2A, ensuring vendor neutrality.

The MCP-A2A relationship is explicitly complementary: MCP governs how an agent uses tools, A2A governs how agents talk to each other. In practice, a well-architected agentic system uses both—MCP connections to databases, APIs, and execution environments, and A2A connections to other agents in a multi-agent workflow. The risk is fragmentation: if competing agent communication protocols emerge (and several are being proposed), the interoperability promise of A2A could be diluted before it fully matures.

Agent Identity Standards

Agent identity is the most contested and least mature layer of the agentic stack. Multiple initiatives are converging from different directions. Microsoft Entra Agent ID extends Azure's enterprise identity platform with OAuth 2.0/2.1 flows specialized for agent authentication, including Federated Identity Credentials (FIC) that allow agents to authenticate using tokens from external identity providers. OpenAgents' AgentID, announced in February 2026, takes a decentralized approach using W3C DIDs, challenge-response authentication, and JWT tokens—essentially a cryptographic passport for AI agents. The IETF's AIMS draft (Agent Identity Management System, published March 2, 2026) attempts to compose existing standards—WIMSE, SPIFFE, and OAuth 2.0—into a unified framework. And NIST's concept paper (February 2026) is exploring how identity and authorization practices can be adapted for agent use in enterprise settings.

The fundamental tension is between authentication (proving an agent is who it claims to be) and authorization (controlling what an agent can actually do). Most current solutions handle the first well but leave the second largely unsolved. As one researcher put it, AI agent stacks in 2026 are at the "log into your bank" phase—authentication works, but transaction monitoring and fine-grained authorization are missing entirely.

Agent Payment Infrastructure

Agent payments have evolved from theoretical to operational faster than most observers predicted, though the landscape is bifurcated. On the enterprise/fiat side, Stripe launched its Machine Payments Protocol (MPP) alongside the Tempo blockchain (built with Paradigm). MPP introduces session-based payments where an agent authorizes a spending limit upfront, then streams micropayments against that session—aggregating and settling in bulk rather than requiring individual on-chain transactions per API call. Stripe has invested over $1.1 billion in this infrastructure, including the acquisition of stablecoin platform Bridge. On the crypto-native side, the x402 protocol enables permissionless USDC micropayments on Coinbase's Base network, with each API call settled individually on-chain. Over 35 million transactions have flowed through x402 as of March 2026.

Google's Agent Payments Protocol (AP2), announced in September 2025, takes a different approach centered on signed "mandates"—structured authorization documents that include explicit user consent, spending limits, and service descriptions. Coinbase launched Agentic Wallets in February 2026, providing wallet infrastructure designed specifically for AI agents. And Visa has signaled plans to support AI-powered commerce by holiday 2026, targeting millions of agent-initiated transactions.

Assessment: The payment layer is bifurcating along a familiar line—Stripe/Visa for regulated, high-compliance environments; crypto rails for permissionless, global micropayments. The most likely outcome is convergence: Stripe's MPP already supports both fiat and USDC settlement, and x402's simplicity makes it attractive even to traditionally fiat-only businesses. The critical missing piece is a universal agent spending authorization standard that works across both paradigms.
Section 4

Enterprise vs. Open Web

Divergent Requirements

The infrastructure requirements for agentic AI diverge sharply depending on whether agents operate within enterprise boundaries or on the open internet. Enterprise deployments benefit from controlled environments: known identity providers (Microsoft Entra, Okta), defined permission boundaries, existing audit infrastructure, and internal network segmentation. Agents operating within Salesforce's Agentforce, ServiceNow's workflow automation, or Microsoft Copilot Studio inherit the security and governance postures of their host platforms. IBM's research shows that organizations using standardized protocols reduce integration time by 60–70% compared to custom development—a 200-integration enterprise might need only 30 MCP connections instead of 200 custom API clients.

Open-web agents face fundamentally harder problems. They encounter untrusted content (potential prompt injection vectors), unknown counterparties (requiring portable identity verification), variable rate limits, inconsistent API schemas, and no central authority to mediate disputes. The trust model inverts: in the enterprise, trust is assumed and exceptions are managed; on the open web, distrust is the default and trust must be established per-interaction.

Platform Company Positioning

The major platform companies are positioning aggressively, each from their existing strengths. AWS offers Amazon Bedrock as a managed multi-model runtime with Agents for Bedrock (multi-step task execution), Knowledge Bases (RAG), and Guardrails (safety and governance), all deeply integrated with AWS infrastructure. Its approach is model-agnostic, supporting Anthropic Claude, Meta Llama, Mistral, Cohere, and Amazon's own Titan models. Microsoft Azure leads with OpenAI model integration, Entra Agent ID for identity, and Copilot Studio for business-user agent creation—the strongest play for enterprises already in the Microsoft ecosystem. Google Cloud differentiates on openness and interoperability, promoting its open-source Agent Development Kit (ADK) and the A2A/MCP protocols within Vertex AI Agent Builder, plus the AI Agent Marketplace for discovering and purchasing third-party agents.

Salesforce has embedded agentic capabilities across its entire platform through Agentforce, Einstein GPT, and Data Cloud—positioning itself as the agentic layer for customer-facing business processes. Shopify is making a particularly interesting bet: as AI agents increasingly act as shoppers rather than humans browsing on phones, Shopify's commerce infrastructure is being rebuilt to serve non-human buyers, with agent-optimized APIs, structured product data, and autonomous checkout flows. NVIDIA launched an enterprise AI agent platform at GTC 2026 with 17 initial adopters including Adobe, Salesforce, and SAP, focusing on the compute and inference infrastructure that agents require at scale.

Section 5

Security, Trust, and Governance

Novel Attack Surfaces

Autonomous agents introduce attack surfaces that have no direct precedent in traditional software security. The most fundamental is indirect prompt injection: malicious instructions embedded in web content, emails, documents, or database records that an agent ingests during normal operation. Because large language models process instructions and data as tokens in the same context window, prompt injection is a structural feature of the architecture—not a bug that can be patched. Research from Q4 2025 showed that indirect attacks often required fewer attempts to succeed than direct attacks, making external data sources the primary risk vector heading into 2026.

Agent impersonation becomes possible in any system where agent identity is weakly authenticated. Without cryptographic identity verification, a malicious agent could claim to represent a trusted service, extract sensitive data, or initiate unauthorized transactions. Credential theft and lateral movement are amplified when agents hold OAuth tokens, API keys, or wallet credentials that provide access to multiple services—compromising one agent can cascade across an entire toolchain. Cascading failures in multi-agent systems represent a systemic risk: if agents delegate to other agents (as A2A enables), a single compromised or hallucinating agent can propagate errors through a chain of dependent agents before any human observes the problem.

Governance Frameworks and Containment

The governance landscape is rapidly evolving, though implementation lags behind awareness. NIST's AI Agent Standards Initiative (February 2026) is the most significant institutional effort, establishing a framework for agent identity, authorization, and secure interoperability. The Agentic AI Foundation under the Linux Foundation is developing protocol-level governance for MCP and A2A. At the enterprise level, Cisco's State of AI Security 2026 report documents the expanding threat landscape, and Microsoft's security team published research in March 2026 on detecting and analyzing prompt abuse patterns.

The most concerning finding from industry surveys is the governance-containment gap: while a majority of enterprises (58–59%) report monitoring or human oversight of agent actions, far fewer (37–40%) have implemented actual containment controls—purpose-binding constraints that limit what an agent can do, or kill-switch capabilities that can halt agent execution immediately. This gap means that many organizations can observe an agent misbehaving but cannot stop it in real time.

Practical recommendations emerging from the security research community include: treating all external data as untrusted input with explicit sanitization before agent ingestion; implementing defense-in-depth across the full agent interaction chain (prompts, retrieval steps, tool calls, outputs); requiring human-in-the-loop approval for irreversible or high-value actions; maintaining immutable audit logs of all agent actions with sufficient context for post-hoc investigation; and designing agent permission models on the principle of least privilege, with time-bounded and scope-limited authorization tokens.

Open Question: How should agent actions be made reversible? Unlike traditional software transactions that can be rolled back, agent actions in the real world—sending an email, executing a trade, publishing content—often cannot be undone. The industry lacks a standard pattern for agent action reversibility, and this may prove to be one of the most difficult design challenges of the agentic web.
Section 6

Competitive Dynamics

Horizontal Infrastructure vs. Vertical Applications

The competitive landscape is stratifying into three tiers. At the foundation layer, the hyperscalers (AWS, Azure, GCP) and model providers (OpenAI, Anthropic, Google DeepMind) are building horizontal infrastructure—runtimes, protocols, identity systems, and marketplaces that aim to be the substrate on which all agents run. This is a classic platform play, and the dynamics are familiar: winner-take-most at each layer, with competition centered on ecosystem breadth, developer experience, and enterprise trust.

At the middleware layer, companies like LangChain, CrewAI, and a growing cohort of startups are building the orchestration, observability, and tooling that sits between foundation models and applications. LangChain's 47M+ downloads give it a significant ecosystem moat, but the middleware layer is vulnerable to platform encroachment—both OpenAI (via the Agents SDK) and Google (via ADK) are building competing capabilities natively into their platforms.

At the application layer, vertical agent companies are building domain-specific solutions: Agentforce for CRM workflows, Shopify's agent-commerce infrastructure, specialized agents for legal, healthcare, financial, and engineering use cases. The application layer is where most venture capital is flowing, but defensibility depends on proprietary data advantages and workflow integration depth rather than technology alone.

Capital Flows

The funding landscape reflects the market's conviction that the agentic web is not speculative. Agentic AI companies have raised $20.8 billion cumulatively across 530 funded companies as of March 2026. In 2025 alone, $5.99 billion was raised across 213 rounds—a 30% increase year-over-year. The "agentic" label commands a measurable premium: seed rounds for agentic startups are valued 40% higher than equivalent "generative AI tools" startups.

Notable recent infrastructure-focused rounds include Exa ($85M Series B for AI-native search/retrieval), Spangle AI ($15M Series A for agentic commerce infrastructure), and Trace ($3M seed, YC W25, for agent workflow orchestration). At the larger end, Stripe's $1.1B+ investment in agent payment infrastructure (including the Bridge acquisition and Tempo blockchain development) represents the single largest capital commitment to any specific layer of the agentic stack.

Emerging Moats

Three types of competitive moats are forming. Protocol network effects: MCP's 10,000+ servers create a self-reinforcing ecosystem—more servers attract more agents, which attract more server developers. Displacing MCP would now require not just a technically superior protocol but a migration path for the entire ecosystem. Data and context advantages: agents that accumulate user preferences, workflow history, and domain knowledge over time become increasingly difficult to replace—Salesforce's Data Cloud and Shopify's merchant data are examples of this dynamic. Trust and compliance certification: in enterprise markets, the first platforms to achieve SOC 2, HIPAA, and FedRAMP compliance for agentic workloads will have a significant head start, as competitors must replicate not just technology but audit trails, governance processes, and regulatory relationships.

Section 7

Key Open Questions

As the agentic web enters its institutionalization phase, several questions will shape its trajectory over the next 12–24 months.

Will identity converge? The proliferation of agent identity approaches (Microsoft Entra, OpenAgents DID, IETF AIMS, NIST frameworks) risks creating the same fragmentation that plagued federated identity on the human web. The IETF's AIMS draft, which composes existing standards rather than inventing new ones, may represent the most viable convergence path—but enterprise inertia around Microsoft Entra could create a de facto standard regardless of formal standardization.

Can prompt injection be structurally mitigated? If prompt injection remains a fundamental feature of how LLMs process context, the agentic web may require an entirely new architectural pattern—perhaps separating the "reasoning" context from the "data" context at the model level, or routing all external data through deterministic sanitization layers. No satisfactory solution has emerged, and this remains the most significant technical risk.

Who captures the economics? As agents mediate more internet transactions, the value chain shifts from human attention (advertising) to agent capability (tool access, compute, data quality). This has profound implications for business models: websites that optimize for agent consumption over human browsing may capture disproportionate value, while those that remain human-only risk invisibility in an agent-mediated web.

Will the open web survive the agentic transition? The traditional web's openness was partly a function of its human-readable nature: anyone could view source, follow links, and access content. Agent-mediated access introduces new gatekeeping possibilities—proprietary tool protocols, closed agent marketplaces, and paywalled data access. Whether the agentic web remains open or consolidates around a few proprietary platforms depends on decisions being made now, in the protocol and governance design phase.

How fast does regulation arrive? The EU AI Act's provisions on high-risk AI systems will intersect with agentic web infrastructure in ways not yet fully mapped. Agent actions that involve financial transactions, healthcare decisions, or employment recommendations may trigger compliance requirements that reshape which architectures are viable. NIST's proactive engagement is encouraging, but regulatory frameworks typically lag technical deployment by years—a gap that creates both opportunity and risk.
