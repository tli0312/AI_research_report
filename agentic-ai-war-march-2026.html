Market Intelligence Brief

The Agentic AI War

A strategic analysis of the four-way battle for the autonomous agent platform layer — who's winning, how, and what it means for enterprise adoption.

March 23, 2026 — Prepared for Tong Li

Executive Summary

The agentic AI market has shifted from a benchmarks race to a platform architecture war. As of March 2026, four major players — Anthropic, Microsoft, Google, and OpenAI — have each placed fundamentally different bets on how autonomous agents should execute, persist, and earn enterprise trust. Anthropic is betting on local-first sandboxing with phone-to-desktop handoff. Microsoft is leveraging its unmatched enterprise distribution through Microsoft 365 and Azure. Google is pursuing a universal multimodal agent that follows users across devices and form factors. OpenAI is scaling cloud-native async execution with the deepest consumer install base.

The central finding: Anthropic currently leads the Enterprise Trust battle through its local VM sandboxing architecture, OS-level isolation primitives, and zero-data-retention commitments — but Microsoft's distribution advantage and sovereign data processing rollout make it the most formidable long-term competitor for regulated industries.

The Combatants & Their Primary Offerings

Anthropic

Code / Cowork / Dispatch

Claude Code is a terminal-native agentic coding tool that runs locally with OS-level sandboxing (Linux bubblewrap, macOS seatbelt). It reduces permission prompts by 84% inside sandboxed environments while maintaining strict filesystem and network isolation.

Cowork (research preview, Feb 2026) extends Claude Code's power to knowledge workers. It runs a full Linux VM locally on the user's machine with multi-layered security: VM + sandbox + seccomp + network allowlists. Supports plugins, MCP connectors, and persistent file access.

Dispatch (March 17, 2026) introduces phone-to-desktop remote control — users can initiate, monitor, and steer desktop agent tasks from their mobile device, as long as the desktop app is running.

Philosophy: "Your data never leaves your machine." Agent runs in a local sandbox; only API calls and package registry traffic leave the VM.

Microsoft

Copilot / AutoDev / Agent 365

Microsoft 365 Copilot (Wave 3) has evolved from a chat assistant to an agentic layer embedded across Word, Excel, PowerPoint, and Teams. "Agent Mode" enables iterative, multi-step document collaboration. Notably, Wave 3 includes Copilot Cowork — built in partnership with Anthropic using Claude.

AutoDev (launched May 2025) is a fully autonomous coding agent that manages end-to-end software projects: file editing, testing, git operations, and cloud deployment inside Docker containers. Achieves 91.5% on HumanEval.

Agent 365 (GA May 2026) provides a unified governance platform to observe, manage, and secure agents across the enterprise.

Philosophy: "Meet enterprises where they already live." Agents embedded in M365 + Azure, governed by existing IT infrastructure.

Google

Gemini / Astra / Mariner

Gemini 2.0+ serves as the foundational model layer, with native multimodal reasoning, tool use, and agentic planning capabilities. Gemini 2.5 Computer Use enables browser-driving tasks through predefined actions.

Project Astra is a universal AI assistant prototype with real-time vision, audio, and memory. Available on Android/iOS via Gemini Live, with smart glasses hardware expected 2026. Crucially, Astra maintains context across device switches (phone → glasses) without interruption.

Project Mariner is a browser-native agent built on Gemini 2.0 that can run up to 10 parallel tasks — research, bookings, purchases — simultaneously within Chrome and Workspace.

Philosophy: "One agent, every surface." From phone camera to browser to smart glasses, the agent follows you and never drops context.

OpenAI

ChatGPT Agent / Codex

ChatGPT Agent (formerly Operator, deprecated Aug 2025) combines Operator's browser interaction, Deep Research's synthesis, and ChatGPT's conversational fluency. Powered by CUA (Computer-Using Agent) model with GPT-4o vision + reinforcement learning. Rolling out to Pro, Plus, Team, Enterprise.

Codex (Jan 2026) is a cloud-based autonomous coding agent. Each task runs in its own isolated cloud sandbox pre-loaded with your repository. Internet is disabled during execution for security. Powered by codex-1 (optimized o3). Fully asynchronous — assign a task, close your laptop, return to a completed PR.

Philosophy: "Fire and forget." Cloud-native async execution at scale; the agent works while you sleep.

The Three Pillar Analysis

Pillar 1 — Local vs. Cloud Execution

Player Execution Model Architecture Details
Anthropic LOCAL-FIRST Cowork runs a full Linux VM on the user's machine using native virtualization. Inside the VM: sandbox + seccomp + network allowlist. Only Anthropic API calls and package registry traffic leave the machine. Claude Code uses OS-level primitives (bubblewrap/seatbelt) for filesystem and network isolation without containers. Code and data stay local.
Microsoft CLOUD HYBRID M365 Copilot processes within Microsoft's cloud service boundary, honoring existing M365 data protection and access controls. New in 2026: sovereign data processing rolling out to 15 countries, allowing local in-region processing. AutoDev runs in Docker containers but predominantly cloud-hosted via Azure. Agent 365 provides centralized governance.
Google CLOUD Gemini, Astra, and Mariner all run in Google's cloud infrastructure. Mariner operates within a Chrome extension context, interacting with the user's browser but reasoning server-side. Astra processes multimodal input (camera, audio) through cloud endpoints. Google offers confidential computing encryption but execution remains cloud-dependent.
OpenAI CLOUD ChatGPT Agent uses its own cloud-hosted browser instance. Codex runs each task in an isolated cloud sandbox with internet access disabled during execution. Codex CLI exists as a local terminal tool, but the heavy lifting (code generation, test execution) happens in OpenAI's cloud. Security model: isolation via containerization, not local execution.

Analysis: Anthropic is the only major player that has committed to a local-first execution model for its flagship consumer/prosumer agent product. This is an architecturally bold bet — it trades the scalability advantages of cloud execution for a fundamentally stronger privacy and trust story. Microsoft's hybrid approach (cloud-native with sovereign processing options) is the pragmatic enterprise play. Google and OpenAI remain cloud-dependent, relying on container isolation and access controls rather than locality for their security narrative.

Pillar 2 — Synchronous (Chat) vs. Asynchronous (Background) Task Handling

Player Model Capabilities
Anthropic SYNC with async emerging Cowork is primarily synchronous — the user watches the agent work in a live desktop session. Dispatch adds a remote-monitoring layer (phone-to-desktop), but the agent still requires the desktop app to be running. Claude Code operates in foreground terminal sessions. True background/async execution is not yet a core offering.
Microsoft HYBRID M365 Copilot Agent Mode supports both interactive collaboration and autonomous background tasks. Copilot Studio enables building agents that trigger on events and run without user presence. However, most current UX is still chat-initiated and foreground. Agent 365 governance implies an infrastructure designed for persistent background agents.
Google HYBRID Project Mariner can run up to 10 parallel tasks simultaneously — a form of concurrent async. However, tasks are initiated within an active browser session. Astra maintains persistent memory across interactions, enabling pick-up-where-you-left-off continuity. Not yet a true "fire and forget" background model.
OpenAI ASYNC LEADER Codex is fully asynchronous by design: assign a coding task, close your laptop, return to a completed pull request. Each task runs in its own cloud sandbox independently. ChatGPT Agent combines real-time chat with background task execution. Developer API offers explicit "background mode" for long-running tasks without timeouts. Most mature async story in the market.

Analysis: OpenAI has the clearest async-first vision, particularly through Codex. The "assign and walk away" model is a genuine workflow innovation for developers. Anthropic's sync-first model is a natural consequence of its local-first architecture — a local VM needs the host machine running. Dispatch is a bridge solution (remote-monitor from phone), but Anthropic hasn't yet shipped a true cloud-based background agent. This is arguably Anthropic's biggest architectural gap. Microsoft and Google occupy the middle ground with hybrid approaches that are evolving toward more autonomous background execution.

Pillar 3 — Device-to-Device Handoff Capabilities

Player Rating Handoff Architecture
Anthropic EMERGING Dispatch (March 2026) is Anthropic's first device handoff feature: initiate or monitor desktop agent tasks from your phone. However, this is specifically a phone → desktop remote-control pattern, not a symmetric multi-device context transfer. The desktop must remain on and running. Context lives on the local machine, not in a cross-device cloud layer.
Microsoft STRONG M365's existing cross-device infrastructure (Windows, mobile apps, web) provides a natural handoff layer. Copilot context persists in Microsoft Graph, accessible from any authenticated device. However, agent state handoff (handing off a mid-execution agentic task between devices) is not yet a first-class feature — the graph stores data context, not live agent state.
Google LEADER Project Astra is the gold standard for device handoff. Users can switch between Android phone, prototype smart glasses, and desktop without losing conversational or task context. Astra's persistent memory layer abstracts away the device boundary entirely. Combined with Mariner (browser) and Gemini Live (mobile), Google is building the most seamless cross-surface agent. Target: a "Universal Agent" by late 2026.
OpenAI MODERATE ChatGPT's cross-platform availability (web, mobile, desktop) provides basic conversation continuity. Codex tasks persist in the cloud and can be checked from any device. However, there is no explicit device-handoff feature for live agent sessions — no equivalent to Dispatch or Astra's context-preserving device switching.

Analysis: Google's lead in device handoff stems from its hardware + software integration strategy (Android, glasses, Workspace, Chrome). Astra's ability to maintain context across completely different form factors — from a phone camera recognizing a broken appliance to a browser purchasing the replacement part — is the most compelling cross-device story. Anthropic's Dispatch is clever but architecturally constrained by the local-first model: the desktop is the bottleneck. Microsoft has the infrastructure for handoff but hasn't yet made it agentic. OpenAI has largely ignored this dimension in favor of async execution.

Verdict: Who Is Winning Enterprise Trust?

Anthropic leads on architecture. The local VM + multi-layer sandbox model is the strongest technical answer to "where does my data go?" that any vendor offers today. When the agent runs on your machine, inside an isolated VM, with network traffic restricted to only Anthropic's API and package registries, the attack surface is fundamentally smaller than any cloud-hosted alternative. The 84% reduction in permission prompts through sandboxing demonstrates that security and usability aren't zero-sum. For organizations where data sovereignty is non-negotiable — healthcare, legal, defense — Anthropic's architecture is the most compelling.

Microsoft leads on governance and distribution. Agent 365 (GA May 2026) gives IT teams a single pane to observe, govern, and secure agents across the entire organization — something no other vendor offers at comparable scale. Microsoft's sovereign data processing rollout to 15 countries, plus its FedRAMP certifications, make it the strongest play for regulated global enterprises that need compliance checkboxes ticked. The existing M365 service boundary and Graph-based access controls mean enterprises don't need new infrastructure.

Google and OpenAI trail on trust narratives. Both rely on cloud-container isolation, which is technically sound but harder to explain to a CISO. OpenAI's Codex approach — disabling internet during execution — is elegant but still means your code runs on OpenAI's servers. Google's confidential computing is a strong technical feature but isn't well-marketed in the agentic context.

Bottom line: If the question is "who has the best security architecture for agentic AI," the answer is Anthropic. If the question is "who will win the most enterprise contracts," the answer is probably still Microsoft — because trust in enterprises is as much about procurement relationships and compliance paperwork as it is about architecture.

Strategic Outlook & Key Risks

Anthropic's risk is that local-first limits scale and async capability. As competitors ship "fire and forget" background agents, Anthropic must answer the question: can a local VM model support true async workflows without the desktop being on? Dispatch is a bridge, not a solution. Revenue growth ($1B → $7B run rate in 2025) and a potential 2026 listing signal confidence, but the product must evolve.

Microsoft's risk is complexity and fragmentation. Between Copilot, AutoDev, Agent 365, Copilot Studio, and the Anthropic-powered Copilot Cowork, the product surface is sprawling. Enterprise buyers may struggle to understand which agent does what. The partnership with Anthropic for Copilot Cowork is telling — it suggests Microsoft's own model capabilities may not be sufficient for the most demanding agentic use cases.

Google's risk is execution and enterprise credibility. Project Astra's device-handoff demo is the most compelling vision of any player, but Google has a history of sunsetting products before they reach maturity. The "Universal Agent" by late 2026 is an ambitious timeline. Google also lacks Microsoft's enterprise sales machine and Anthropic's trust narrative.

OpenAI's risk is trust erosion and commoditization. OpenAI was "the default" for enterprise AI in 2024-2025, but that position is actively eroding. The shift from Operator to ChatGPT Agent (deprecating and re-integrating within 7 months) signals strategic uncertainty. Codex's async model is best-in-class for developers, but the broader consumer/enterprise agent story is less differentiated than competitors'.
